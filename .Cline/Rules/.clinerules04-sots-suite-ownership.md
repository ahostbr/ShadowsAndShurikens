# SOTS Suite Ownership Locks (High-Level)

## Travel  mission flow (authoritative chain)
ProfileShared → UI → MissionDirector

## Save triggers (only these 3)
1) time-based autosave
2) UI-initiated save
3) checkpoint save

## UI authority
- Only SOTS_UI can createpushpopfocus widgets and own input modepausefocus.
- Other plugins request UI via intentspayloads (no direct widget work).

## Tag authority boundary rule
- For cross-plugin runtime actor-state tags (PlayerDragonGuardsPickups that other plugins observe), use SOTS_TagManager APIs.
- Internalprivate tags inside one plugin may bypass TagManager.

## FX pattern
- Systems trigger FX via FXManager one-shots (no UI dependency).

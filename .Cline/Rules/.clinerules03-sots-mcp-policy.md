# SOTS MCP Policy (Single Server)

## Single server rule
- Treat “single server” as: ONE MCP entrypoint started by the host (VSCode/Cline), exposing multiple tool groups.
- Do NOT add multiple MCP servers unless Ryan explicitly asks.

## Canonical entrypoint
- DevTools/python/sots_mcp_server/server.py

## Apply gating
- APPLY over MCP must be hard-gated, OFF by default.
- Only enable APPLY when Ryan explicitly requests (e.g., env SOTS_ALLOW_APPLY=1).

## Safe default behavior
- Prefer read-only tools (list/read/search/report) until an explicit “apply changes” request exists.

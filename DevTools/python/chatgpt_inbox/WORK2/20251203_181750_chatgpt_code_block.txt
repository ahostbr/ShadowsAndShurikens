# DevTools v0.6 – File Mapping (skeleton)

_This file was created by the DevTools v0.6 Phase A pack via write_files.py.
It is intentionally a **skeleton** mapping so we can verify the routing +
write_files pipeline before doing a fully-automatic inventory._

## Purpose

- Capture a high-level view of the current DevTools/python contents in v0.5.
- Group existing tools into the rough v0.6 categories (core, build, edit,
  packs, KEM, FSOTS, BPGen, misc).
- Serve as a human-readable planning doc that later passes and/or a
  generator script can refine into a full per-file table.

## Snapshot Source

- Snapshot: `DEVTOOLS_V00.5_LATEST.zip`
- Root: `DevTools/python`
- Approximate file count at root: ~80 (Python tools, JSON, BATs, logs).

## Categories (initial sketch)

This section is deliberately high-level; it only lists the **most important
and obvious** tools in each category. Later v0.6 passes can either:

- Regenerate this file via a dedicated “mapping generator” script, or
- Extend it manually once the devtools.* package layout is finalized.

---

### core

Core entrypoints, bridge pieces, and hub scripts that likely stay near the
DevTools/python root (or act as the brain of the toolbox):

- `sots_tools.py` – main DevTools menu / hub.
- `sots_tools_config.json` – DevTools configuration.
- `sots_bridge_server.py` – bridge server used by the ChatGPT → DevTools flow.
- `sots_chatgpt_dispatcher.py` – dispatcher that routes [SOTS_DEVTOOLS] packs.
- `sots_pipeline_hub.py` – higher-level pipeline orchestrator.
- `bridge_runner.py` – helper/runner for the bridge.
- `run_devtool.py` – generic “run a tool by name/module” entrypoint.
- `__init__.py` – makes DevTools/python importable as a package.

Core helper utilities (planned for `devtools.core.*`):

- `project_paths.py` – path helpers (project roots, plugin dirs, etc.).
- `cli_utils.py` – shared CLI parsing and menu helpers.
- `log_utils.py` – logging helpers and log file conventions.
- `devtools_header_utils.py` – helpers for `[SOTS_DEVTOOLS]` header parsing.
- `llm_log.py` – tools for logging/inspecting LLM interactions.

---

### build

Build / CI / maintenance helpers that support keeping the project clean and
buildable:

- `analyze_build_log.py` – parses build logs and extracts errors/warnings.
- `ci_checks.py` – grouped CI-style checks for the project.
- `clean_binaries_intermediate.py` – nukes plugin/engine `Binaries/` and
  `Intermediate/` safely.
- `compare_plugin_zips.py` – compares plugin zip snapshots.
- `delete_paths.py` – generic path deletion helper.
- `delete_target.py` – targeted delete helper (single file or tree).
- `uplugin_utils.py` – helpers for working with `.uplugin` files.
- `run_unreal_project.py` – convenience wrapper to launch the project/UE.

---

### edit

Batch edit and search utilities:

- `ad_hoc_regex_search.py` – run arbitrary regex searches across the tree.
- `mass_regex_edit.py` – apply regex-based replacements from a config.
- `symbol_refs.py` – symbol/reference scanning helper.
- `tag_usage_scan.py` – scans for gameplay tag usage patterns.
- `inject_license_header.py` – injects or verifies license headers.
- `scan_todos.py` – scans for TODO / FIXME comments.

---

### packs / routing

Helpers for DevTools “packs” and routing:

- `apply_json_pack.py` – applies JSON DevTools packs to files.
- `validate_sots_pack.py` – validates a DevTools pack’s structure/content.
- `check_tag_spine.py` – tag spine checker/report (bridges with TagManager).
- `write_files.py` – parses `[SOTS_DEVTOOLS]` + `=== FILE: ... ===` blocks
  and writes them into the project tree.
- `sots_plugin_modules.json` – plugin/module mapping used by packs/tools.
